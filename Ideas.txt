Picture is made from Pixels where every Pixel can have an adjacent Pixel
The shape of a Pixel is usually rectangle
A rectangle have 4 sides

The input is a picture
The output is a picture
There are multiple input and every input has exactly one output
There is an unknown picture transformation between input and output

Transformations on a picture:
 - mirror
 - crop
 - inflate / shrink

There can be objects on picture which has features
 - object size: x / y 
 - pixel count

These features can be part of the transformation

The goal of the program is to find transformation chain to get from input to output.

Input can be brak down to sub objects and the transformation and selection method can be applaid on these objects.

input -> transform(input) == output
or
input | getObjects | transform == output

properties of 1D object (list): size, firstElem, lastElem, firstIterator, lastIterator
properties of 2D object (plane): size, firstElem, lastElem, firstIterator, lastIterator


transformations: sub

#include <iostream>
#include <string>

template<typename T, unsigned N>
T* first(T (&input)[N])
{
    return &input[0];
}

template<typename T, unsigned N>
T* last(T (&input)[N])
{
    return &input[N - 1];
}

template<typename T>
T* next(T* input)
{
    return ++input;
}

template<typename T>
T* prev(T* input)
{
    return --input;
}

template<typename T>
T& get(T* input)
{
    return *input;
}

int main()
{
    char input[] = { 'A', 'B', 'C' };
    char output[] = { 0, 0, 0 };
    
    char* inputIt = input;
    char* outputIt = first(output);
    
    for (unsigned i = 0; i < sizeof(input); ++i) {
        get(outputIt) = get(inputIt);
        inputIt = next(inputIt);
        outputIt = next(outputIt);
    }
    
    std::string resI(input, input + sizeof(input) / sizeof(input[0]));
    std::string resO(output, output + sizeof(output) / sizeof(output[0]));
    
    std::cout << resI << std::endl;
    std::cout << resO << std::endl;
    
    return 0;
}
